Log in API (аутентификация с помощью JWT)

  • POST - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com/api/login
  • В теле запроса идет следующий словарь 
  • {
    "API_Key": "JWT ключ",
    "user_id": TG ID
}
  • Можно получить 2 вида словаря
    • Успех 
      • {
    "answer": "Проверка прошла успешно телеграмм успешно привязан к аккаунту",
    "status_code": "authorized"
}
    • Провал 
      • {
    "answer": "Проверка провалена, телеграм не привязан к аккаунту",
    "status_code": "not authorized"
}

Log in API (аутентификация с помощью логина и пароля)

  • POST - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com/a_api/login
  • В теле запроса данные в следующем формате 
  • {
    "login": "login",
    "password": "passwd"
}
  • Можно получить 3 вида словаря
    • Успех
      • {
    "jwt": "Тут будет JWT ключ",
    "login": "success"
}
    • Провал(не правильный логин) 
      • {
    "login": "There is no such login"
}
    • Не правильный пароль
      • {
    "login": "Incorrect login or password"
}

Posts API - получение данных всех постов

  • GET - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com//posts
  • Вместо - тот ключ который принадлежит юзеру
  • В теле запроса ничего нет
  • Варианты ответа
    • Успех
      • [
    [
        1,
        1,
        "katana",
        "ЧвК Редан?О_О ryodan??",
        "Название: Геней Рёдан etc.
    ],
    [
        2,
        1,
        "katana",
        "ZXC???",
        "100-7???????"
    ]
]
    • Провал 
      • {
    "denied": "Отказано в доступе"
}

One post API - получение данных конкретного поста

  • GET - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com//posts/
  • Вместо - тот ключ который принадлежит юзеру
  • Вместо - конкретный id поста
  • Варианты ответа
    • Успех
      • {
    "comment": [
        [
            1,
            1,
            2,
            "rukati",
            "Очень крутой пост спасибо большое"
        ]
    ],
    "post_data": [
        [
            1,
            1,
            "katana",
            "ЧвК Редан?О_О ryodan??",
            "Название: Геней Рёдан/Призрачная Труппа etc
        ]
    ]
}
    • Несуществующий пост 
      • {
    "denied": "Такого поста нет"
}
