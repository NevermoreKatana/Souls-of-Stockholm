Log in API (аутентификация с помощью JWT)

  • POST - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com/api/login
  • В теле запроса идет следующий словарь
  • { "API_Key": "JWT ключ", "user_id": TG ID }
  • Можно получить 2 вида словаря
    • Успех
      • { "answer": "Проверка прошла успешно телеграмм успешно привязан к аккаунту", "status_code": "authorized" }
    • Провал
      • { "answer": "Проверка провалена, телеграм не привязан к аккаунту", "status_code": "not authorized" }

Log in API (аутентификация с помощью логина и пароля)

  • POST - запрос
  • Запрос делается по адресу https://souls-of-stockholm.onrender.com/a_api/login
  • В теле запроса данные в следующем формате
  • { "login": "katana", "password": "12345678" }
  • Можно получить 3 вида словаря
    • Успех
      • { "jwt": "Тут будет JWT ключ", "login": "success" }
    • Провал(не правильный логин)
      • { "login": "There is no such login" }
    • Не правильный пароль
      • { "login": "Incorrect login or password" }
